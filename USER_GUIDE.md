# wx-auto 微信自动化系统 · 用户使用须知

> wechatapi 已完全封装到后端流程中，你 **不需要** 了解 wechatapi 文档、注册、调试或调用细节。Token 由管理员手动发放，直接配置即可使用。

---

## 快速使用

1. **联系管理员** 获取 wechatapi token + appId
2. **告诉 Agent**：「配置微信 token 为 XXXX，appId 为 XXXX」（Agent 会自动调 API 设置）
3. **扫码登录**：Agent 生成二维码 → 微信扫码 → 确认登录
4. **测试**：发一条消息给文件传输助手，确认 AI 能回复

全程 Agent 引导，无需手动部署或编程。

## 如何唤醒微信 AI

### 渠道 A：0913 桥接（主渠道，全功能）

在 Hermes 对话中，提及以下 **任意关键词** 即自动加载微信技能：

| 关键词 | 示例 |
|--------|------|
| `wx` | 「帮我查wx最近的消息」|
| `微信` | 「用微信给张三发个消息」|
| `微信自动化` | 「开启微信自动化」|
| `wx-auto` | 「wx-auto 配置怎么看」|
| `/wx` | 直接输入 `/wx` 加载微信技能 |

### 渠道 B：iLink Bot（个人 AI 对话，扫码即用）

无需 Deepsee 服务器，直接在 Hermes 终端接入：

```bash
hermes gateway setup  # 选择 Weixin/WeChat → 扫码登录
```

- 私聊模式：只有授权的联系人才可以对话
- 授权：`hermes pairing approve weixin <用户ID>`
- 限制：**无法加入普通微信群**，仅私聊

### 自动回复行为

- 群聊中被 @ 时自动触发（0913 渠道）
- 私聊触发规则可配
- 回复范围：投资话题、日常闲聊均可互动，仅违法内容和系统配置指令除外
- 回复风格：直接自然，像同事聊天

### AI 每天都在变聪明

系统内置 **双环路自我进化**：
- **每天 23:00**：复盘当日微信对话，发现生硬/答非所问 → 自动调整提示词规则
- **每周一**：审计知识库健康度 + 回顾预测准确率 → 校准判断力

你的 AI 越用越懂你。

## 风险警告 ⚠️

### 封号风险（最重要）

| 风险行为 | 说明 |
|---------|------|
| 高频群发/回复 | 每分钟超过 5 条消息对话，系统会触发风控 |
| 群发广告/营销内容 | 含链接、二维码、诱导分享内容触发高频封号 |
| 批量加好友 | 短时间大量添加联系人会被封禁 |
| 自动化回复太像机器人 | 重复话术、无上下文的机械回复容易触发检测 |
| 异地/非常规登录 | 频繁切换设备/IP 或非国内 IP 登录会触发安全验证 |

**建议**：
- 消息间隔保持 2-3 秒以上
- 群发控制在每天 100 条以内
- 不使用微信自动化进行营销、广告、骚扰
- 不使用虚拟号、接码平台注册的微信号

### 账号注册资质要求

**只使用本人或公司实名认证的微信号**：

- ❌ 不接受虚拟号段注册的微信（170/171/162 等）
- ❌ 不接受二手购买的微信号（历史行为不可控，容易连带封号）
- ✅ 推荐使用主微信号或企业微信
- ✅ 微信号需要绑定银行卡完成实名认证
- ✅ 注册时间 > 6 个月的账号更安全

### 隐私与合规

- 自动化工具会读取微信消息，**不要用涉及个人隐私的主号**
- 企业使用建议单独注册工作微信号
- 所有消息记录存储在后端数据库，请做好安全管理

## 使用技巧

1. **一句话搞定一件事**：「把王总的备注改成'重要客户-王总'」「检查微信在线状态」
2. **复杂任务拆着说**：「先查一下最近三天的消息，然后帮我总结成日报」
3. **提到具体名称**：说「给张三发消息」而不是「给那个人发消息」，准确率更高
4. **状态查看**：说「微信在线吗」Agent 会检查在线状态
5. **善用 /wx**：输入 `/wx` 快速加载微信技能，后续对话直接操作

## 技术架构（了解即可）

```
你 ──(说话)──► Hermes Agent ──HTTP──► Deepsee 服务器(:8001) ──HTTP──► wechatapi.net
                                             │
                                        [SQLite 数据库]
```

- **Hermes Agent**：你的 AI 对话界面，理解你的需求并发起微信操作
- **Deepsee**：后端服务器，封装了所有微信协议细节
- **wechatapi.net**：底层 iPad 协议通信层（你不需要碰）

## 故障排查

| 现象 | 可能原因 | 解决 |
|------|---------|------|
| Agent 说微信离线 | token 过期或设备注销 | 发「重新登录微信」按步骤操作 |
| 消息发不出去 | 频率超限 | 暂停 5 分钟再试 |
| 联系人找不到 | 微信号/备注未同步 | 发「刷新联系人列表」|
| 回复很奇怪 | 回调未绑定 | 发「重新绑定微信回调」|
| 完全没反应 | token 失效 | 联系管理员获取新 token 并配置 |

---

**一句话总结：拿到 token → 交给 Agent 配置 → 扫码登录 → 开口说话就行。**
